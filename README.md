# JWT Token Fetcher & Bot Script Setup

## Overview
This system uses external JWT APIs to fetch tokens, avoiding direct Garena API rate limiting. Tokens are fetched once and stored in `tokens.json`, then used by `BoT.py` to process map codes.

## Files
- **jwt_fetcher.py** - Fetches JWT tokens from external APIs and stores them in `tokens.json`
- **BoT.py** - Terminal script that uses tokens from `tokens.json` to process map codes
- **xMaSrY.txt** - Account list in format `uid:password`
- **tokens.json** - Generated by jwt_fetcher.py, contains all successful JWT tokens
- **failed_accounts.txt** - Generated by jwt_fetcher.py, contains accounts that failed to fetch tokens

## Quick Start

### Step 1: Fetch Tokens
```bash
py jwt_fetcher.py
```

This will:
- Read all accounts from `xMaSrY.txt`
- Try each account against 4 external JWT API endpoints
- Use exponential backoff for rate-limited accounts (429)
- Save successful tokens to `tokens.json`
- Save failed accounts to `failed_accounts.txt`
- Display a summary report

Example output:

### Step 2: Process Map Codes
```bash

# Or interactive mode
py BoT.py

```

## How It Works

### Token Fetching Flow (jwt_fetcher.py)
1. **Load Accounts** → Read uid:password from xMaSrY.txt
2. **Try API Endpoints** → For each account, try 4 external JWT APIs in sequence
3. **Retry Logic** → On 429 rate limit: wait 1s, 2s, 4s (exponential backoff)
4. **Save Results** → successful tokens → tokens.json, failed accounts → failed_accounts.txt

### Token Usage Flow (BoT.py)
1. **Load Tokens** → Read tokens.json (no network calls)
2. **Process Codes** → For each map code, use tokens to make requests
3. **Background Refresh** → Check for new tokens.json every hour

## Features

### jwt_fetcher.py
✅ Parallel API endpoint fallback (4 APIs tried in sequence)
✅ Exponential backoff for rate limiting (429)
✅ Network error handling with retry
✅ Beautiful progress logging
✅ Detailed success/failure reports
✅ Skips accounts where token field is empty
✅ JSON output for easy integration

### BoT.py
✅ Uses precomputed tokens (no JWT generation delays)
✅ Terminal/CLI interface (no Telegram dependency)
✅ Command-line arguments: `py BoT.py CODE1 CODE2`
✅ Interactive mode for manual code entry
✅ Async processing with multiple accounts
✅ Hourly token refresh checks (reloads tokens.json if updated)

## Troubleshooting

### All accounts getting 429 errors?
- The accounts may be permanently rate-limited by Garena
- Try using a VPN or different network
- Wait a few hours and try again
- Use fresh account credentials if available

### tokens.json not found?
- Run `py jwt_fetcher.py` first to generate it
- Make sure jwt_fetcher.py completes successfully

### Map codes not working?
- Check that your tokens are still valid (they may expire)
- Run `py jwt_fetcher.py` again to refresh tokens
- Verify the map code format (must be 4+ characters)

### No accounts loaded?
- Check that xMaSrY.txt exists in the same directory
- Format should be: `uid:password` (one per line)
- Remove any commented lines starting with `#`

## Advanced Usage

### Refresh tokens anytime
```bash
py jwt_fetcher.py
```
This regenerates `tokens.json` with fresh tokens. BoT.py will auto-reload it.

### Retry only failed accounts
After running jwt_fetcher.py, it creates `failed_accounts.txt`. You can:
1. Fix the accounts in xMaSrY.txt (update passwords, etc.)
2. Rerun jwt_fetcher.py to try all again
3. Or manually retry just the failed ones

### Use different account list
```bash
# Modify jwt_fetcher.py to use different source, or
# Copy new accounts to xMaSrY.txt and run:
py jwt_fetcher.py
```

## Performance Notes

- **jwt_fetcher.py**: ~1-2 seconds per account (including backoff on rate limits)
- **BoT.py**: Processes codes instantly once tokens are loaded
- With 100 accounts: expect 2-5 minutes total fetch time
- Background token refresh in BoT.py: runs every hour, non-blocking
